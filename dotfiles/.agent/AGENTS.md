# Rules
以下の指示に従って、日本語で簡潔かつ丁寧にタスクの実行、回答をしてください。
- 指示が無い限り大幅にコードを改変しないでください。
- 実装する前に状態を確認してから方針を立てて実装し、最後に実装の不備等を確認し、改善余地があれば、改善してください。
- 実装したコードに単体テストが実装できそうだったらテストもセットで実装してください。(重複するような過度なテストは不要)
- 既存のエンドツーエンドテストなどがあれば、追記してください。
- コメントはWHYが必要な箇所のみにしてください。
- タスクによって適切なスキルを選択して使用してください。
- バグなどを見つけたときはprintデバックなどを積極的に使って自走してみてください。
- gh, gitなどのコマンドを使用して構いませんが、指示が無い限りは`git add`,  `git commit`などのwrite処理は行わずにreadの用途のみで使用してください。
- skillとCLIコマンドを優先的に使ってください。MCPに関しては必要なコンテキストが外部にあったりする場合など、タスクを進めるなかで必要なときにだけ使ってください。
- ユーザーが「skillを作成してください。」のような指示をした場合には`skill-creator`を使ってskillを作成する手伝いをしてください。
- プロジェクト固有のskillなどを作成する場合に現在の作業ディレクトリの `$PWD/.agent` が存在する場合にはそこのskill以下に新しいskillなどを作成してください。
