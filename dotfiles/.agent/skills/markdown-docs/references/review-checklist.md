# マークダウンドキュメント レビューチェックリスト

## 1. 構成・構造

### 見出し
- [ ] 見出し階層が適切（H1→H2→H3の順序）
- [ ] 見出しがスキップされていない（H1の次にH3など）
- [ ] 見出しテキストが具体的で内容を反映
- [ ] 同じ階層の見出しが並列構造

### セクション
- [ ] 論理的な順序で配置
- [ ] 各セクションに適切な分量
- [ ] 冗長なセクションがない
- [ ] 必要な情報が欠落していない

---

## 2. 内容

### 正確性
- [ ] 技術的な記述が正確
- [ ] コード例が動作する
- [ ] コマンド例が正しい
- [ ] バージョン情報が最新

### 網羅性
- [ ] 前提条件が明記
- [ ] インストール手順が完全
- [ ] 設定オプションの説明
- [ ] エラーケースへの対処

### 冗長性
- [ ] 重複した説明がない
- [ ] 不要な前置きがない
- [ ] 過度に詳細な説明がない

---

## 3. 文章

### 文法・表現
- [ ] 誤字脱字がない
- [ ] 文法的に正しい
- [ ] 主語と述語が対応
- [ ] 修飾関係が明確

### 読みやすさ
- [ ] 一文が長すぎない（60文字以内目安）
- [ ] 専門用語に説明あり
- [ ] 適切な接続詞の使用
- [ ] 箇条書きの活用

### 一貫性
- [ ] 文体が統一（です/ます or である）
- [ ] 用語が統一（表記揺れなし）
- [ ] 人称が統一
- [ ] 時制が統一

---

## 4. フォーマット

### マークダウン記法
- [ ] 見出し: `#` の後にスペース
- [ ] リスト: `-` または `*` の後にスペース
- [ ] コードブロック: 言語指定あり
- [ ] リンク: `[text](url)` 形式
- [ ] 画像: `![alt](url)` 形式

### リンク
- [ ] 内部リンクが有効
- [ ] 外部リンクが有効
- [ ] リンクテキストが具体的
- [ ] 相対パスが正しい

### コードブロック
- [ ] 言語が指定されている
- [ ] インデントが適切
- [ ] シンタックスハイライト対応
- [ ] コピー可能（余分な記号なし）

---

## 5. アクセシビリティ

### 画像
- [ ] すべての画像にalt属性
- [ ] alt属性が内容を適切に説明
- [ ] 装飾画像は空のalt

### 構造
- [ ] スクリーンリーダーで読み取り可能
- [ ] 見出しだけで内容把握可能
- [ ] 色だけに依存しない情報伝達

---

## 6. 技術ドキュメント固有

### README
- [ ] プロジェクト名と概要
- [ ] インストール方法
- [ ] 基本的な使い方
- [ ] ライセンス情報
- [ ] 貢献方法（必要に応じて）

### API仕様
- [ ] エンドポイント一覧
- [ ] リクエスト/レスポンス形式
- [ ] パラメータ説明
- [ ] エラーコード一覧
- [ ] 認証方法

### 変更履歴（CHANGELOG）
- [ ] バージョン番号
- [ ] リリース日
- [ ] 変更内容のカテゴリ分け
- [ ] Breaking Changesの明記
