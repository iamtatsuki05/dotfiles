# マークダウンドキュメント レビューチェックリスト

報告形式は SKILL.md「レビューワークフロー Step 3」を正とする。

## 1. 構成・構造

### 見出し
- [ ] 見出し階層が適切（H1→H2→H3の順序、スキップなし）
- [ ] 見出しテキストが具体的で内容を反映
- [ ] 同じ階層の見出しが並列構造

### セクション
- [ ] 論理的な順序で配置
- [ ] 各セクションに適切な分量
- [ ] 冗長なセクションがない
- [ ] 必要な情報が欠落していない

## 2. 内容

### 正確性
- [ ] 技術的な記述が正確
- [ ] コード例・コマンド例が動作する
- [ ] バージョン情報が最新

### 網羅性
- [ ] 前提条件が明記
- [ ] 手順が完全（省略された暗黙のステップがない）
- [ ] エラーケースへの対処

### 冗長性
- [ ] 重複した説明がない
- [ ] 不要な前置きがない

## 3. 文章

### 文法・表現
- [ ] 誤字脱字がない
- [ ] 主語と述語が対応し、修飾関係が明確

### 読みやすさ
- [ ] 一文が長すぎない（60文字以内目安）
- [ ] 専門用語に説明あり
- [ ] 箇条書きの活用

### 一貫性
- [ ] 文体が統一（です/ます or である）
- [ ] 用語が統一（表記揺れなし）
- [ ] 人称・時制が統一

## 4. リンク

- [ ] 内部リンク・外部リンク・見出しアンカーが有効
- [ ] 相対パスが正しい
- [ ] リンクテキストが具体的（「こちら」を避ける）

## 5. アクセシビリティ

- [ ] すべての画像に内容を説明するalt属性（装飾画像は空のalt）
- [ ] 見出しだけで内容の流れを把握可能
- [ ] 色だけに依存しない情報伝達
