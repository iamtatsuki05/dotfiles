# プロンプトエンジニアリング リファレンス

診断時に使う失敗パターン → 対処のマッピングと、改善目標別の戦略。一般的なテクニック(ロール付与、Few-Shot、CoT 等)の解説は載せない。

## よくある失敗パターンと対処法

| 失敗 | 原因 | 対処法 |
|------|------|-------|
| 話題がそれる | システムプロンプトのスコープが広すぎる | 範囲を絞る。「Xに関する質問のみ答えてください」を追加 |
| 回答が冗長 | 長さ制約がない | 「1〜3文で回答してください」を追加 |
| フォーマットが崩れる | フォーマット指定が不明確 | 正確なスキーマと例を提示する |
| ハルシネーション | グラウンディング指示がない | 「提供されたコンテキストの情報のみ使用してください」を追加 |
| トーンが安定しない | トーンが定義されていない | 「フレンドリーかつプロフェッショナルな言葉遣いで」を追加 |
| 正当なリクエストを拒否 | 制約が保守的すぎる | 制約を緩和し、許可されていることを明確化する |
| 指示が無視される | 重要な指示が末尾に埋まっている | 重要なルールをシステムプロンプトの先頭に移動する |

## 目標別の改善戦略

### 正確性を上げる
1. 正しい出力のFew-Shotサンプルを追加する
2. 複雑な推論タスクにはCoTを使う
3. 「慎重に考えてから回答してください」を追記する
4. 複数ステップのタスクをサブタスクに分解する

### 簡潔さを上げる
1. 明示的な長さ制限を追加する（「1〜2文で」）
2. 「簡潔に要点だけ答えてください」を指定する
3. プロンプト自体の冗長な表現を削除する

### フォーマットの一貫性を上げる
1. 出力フォーマットの例を正確に示す
2. フィールドの説明付きJSONスキーマを使う
3. 「このフォーマットから逸脱しないでください」を追加する

### 安全性・ガードレールを強化する
1. 禁止する動作を列挙する
2. フォールバック応答を定義する:「不明な場合は『わかりません』と答えてください」
3. 最終出力の前にモデレーションステップを設ける

### レイテンシ・コストを削減する
1. 単純なタスクはclaude-opusからclaude-haikuに変更する
2. プロンプト内の重複するコンテキストを削除する
3. 主要な制約を保持しながらシステムプロンプトを短縮する
