## 主要テクニック

### ロール + コンテキスト
システムプロンプトの冒頭で明確なペルソナと状況を与える。
```
あなたはAcme社のシニアカスタマーサポート担当です。請求に関する問題のトラブルシューティングを担当します。
```

### 出力の制約
フォーマット・長さ・構造を明示的に定義する。
```
以下のJSON形式のみで回答してください: {"answer": "...", "confidence": "high|medium|low"}
3文を超えないでください。
```

### Few-Shot サンプル
システムプロンプトまたはユーザーメッセージに2〜5個の入出力例を含める。
```
例:
入力: "注文を返金してほしい"
出力: {"intent": "refund_request", "urgency": "high"}

入力: "荷物はどこですか"
出力: {"intent": "tracking_query", "urgency": "medium"}
```

### Chain of Thought (CoT)
最終回答の前にステップバイステップで考えさせる。
```
最終回答を出す前に、ステップバイステップで考えを整理してください。推論を簡潔に示してください。
```

### 否定的指示の明示
してはいけないことを明記する。
```
過度に謝罪しないでください。どうしても必要な場合以外、追加質問はしないでください。
```

## システムプロンプトのパターン

| パターン | 使う場面 |
|---------|---------|
| **ペルソナ + タスク** | 汎用。トーンとスコープを確立する |
| **テンプレート穴埋め** | 構造化出力（フォーム・JSON・表） |
| **デシジョンツリー** | 多段階のルーティングや分類 |
| **制約リスト** | 厳格なガードレールが必要な場面 |
| **RAG（検索拡張）** | 実行時にコンテキストを注入する場面 |

### テンプレート: 構造化出力
```
あなたは[役割]です。ユーザーの入力に対して、以下のスキーマに合致する有効なJSONのみで回答してください:
{
  "field_1": "<型>",
  "field_2": "<型>"
}
マークダウンのコードフェンスや説明文を含めないでください。
```

### テンプレート: 分類
```
ユーザーの意図を [カテゴリ1, カテゴリ2, カテゴリ3] のいずれかに分類してください。
カテゴリ名のみを返し、それ以外は何も返さないでください。
```

## よくある失敗パターンと対処法

| 失敗 | 原因 | 対処法 |
|------|------|-------|
| 話題がそれる | システムプロンプトのスコープが広すぎる | 範囲を絞る。「Xに関する質問のみ答えてください」を追加 |
| 回答が冗長 | 長さ制約がない | 「1〜3文で回答してください」を追加 |
| フォーマットが崩れる | フォーマット指定が不明確 | 正確なスキーマと例を提示する |
| ハルシネーション | グラウンディング指示がない | 「提供されたコンテキストの情報のみ使用してください」を追加 |
| トーンが安定しない | トーンが定義されていない | 「フレンドリーかつプロフェッショナルな言葉遣いで」を追加 |
| 正当なリクエストを拒否 | 制約が保守的すぎる | 制約を緩和し、許可されていることを明確化する |
| 指示が無視される | 重要な指示が末尾に埋まっている | 重要なルールをシステムプロンプトの先頭に移動する |

## 目標別の改善戦略

### 正確性を上げる
1. 正しい出力のFew-Shotサンプルを追加する
2. 複雑な推論タスクにはCoTを使う
3. 「慎重に考えてから回答してください」を追記する
4. 複数ステップのタスクをサブタスクに分解する

### 簡潔さを上げる
1. 明示的な長さ制限を追加する（「1〜2文で」）
2. 「簡潔に要点だけ答えてください」を指定する
3. プロンプト自体の冗長な表現を削除する

### フォーマットの一貫性を上げる
1. 出力フォーマットの例を正確に示す
2. フィールドの説明付きJSONスキーマを使う
3. 「このフォーマットから逸脱しないでください」を追加する

### 安全性・ガードレールを強化する
1. 禁止する動作を列挙する
2. フォールバック応答を定義する:「不明な場合は『わかりません』と答えてください」
3. 最終出力の前にモデレーションステップを設ける

### レイテンシ・コストを削減する
1. 単純なタスクはclaude-opusからclaude-haikuに変更する
2. プロンプト内の重複するコンテキストを削除する
3. 主要な制約を保持しながらシステムプロンプトを短縮する
